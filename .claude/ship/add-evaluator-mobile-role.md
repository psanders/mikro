Newest first. One line per meaningful decision or stage transition.

- 2026-07-01 — Visual verification on iOS Simulator (iPhone 17 Pro) at user's request before sync. Found and fixed 2 real native-crash bugs the build agents' lint/typecheck/unit-test gates never caught: (1) `expo-print` was listed in `app.config.ts`'s `plugins` array but ships no config plugin — Expo tried to load its main entry as a plugin function and crashed with an ESM `SyntaxError` at `expo start`, for ANY platform, not just this build. Fixed by removing it from `plugins` (it's a plain JS API, no native config needed). (2) After that fix, the app still crashed at native launch (`DYLD Symbol not found` in `ExpoPrint.framework`, then in `ExpoFileSystem.framework`) — root cause was version drift across ~8 Expo packages (including `expo` core itself) already present in the repo before this change, which the new `expo-print`/`expo-image-picker` deps' stricter ABI requirements finally exposed. Fixed via `npx expo install --fix` (run twice — first pass converged `expo`/`expo-modules-core`, second pass converged `expo-file-system`/`expo-router`/etc.). That command also swept `typescript` 5→6 monorepo-wide as an unrelated side effect — reverted that specific line in root `package.json` and re-ran `npm install`, verified TS stayed at 5.9.3 and mobile typecheck/lint/tests stayed green. Also added `"types": ["jest"]` to `mods/mobile/tsconfig.json` — a fresh `npm install` stopped auto-resolving `@types/jest` for `tsc`, surfacing ~150 latent type errors in the group-9 test files that had never actually been typechecked cleanly before.
  Rebuilt native 3x (pods + `expo run:ios`) until clean. Then used Maestro (already in the repo for e2e, drives the simulator directly — no macOS Accessibility/Screen Recording permission available in this environment, confirmed blocked) to log in as the seeded Admin user and screenshot: Inicio (evaluator hero card + Más prometedoras list, real seeded data), Cola (filter chips, real queue), application detail (Mikro Score 82/100, risk pill, DETALLES, PROGRESO pipeline), and the Rechazar screen (motivo radio list + comment field) — all render correctly and match the locked Pencil design. Seeded 6 additional example applications via the repo's own `prisma/seed.ts` (already had well-crafted RECEIVED/IN_REVIEW/APPROVED/SIGNED/REJECTED/DRAFT examples with real scores) to get full status/risk-band variety for testing, on top of 2 pre-existing organic dev rows (untouched). No mutations were made through the app itself (only viewed screens, cancelled out of the Rechazar form without confirming). `.env.local` (gitignored, Android-emulator URL) was temporarily pointed at localhost for the iOS run and restored afterward.
  Noted for later: Maestro's text-based `tapOn` selector doesn't reliably match this app's row/link text (had to fall back to point-based taps) — worth a look if task 9.3/9.4's e2e flows get extended, may indicate missing accessibility labels on some pressable rows.
