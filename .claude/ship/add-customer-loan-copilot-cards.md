# Ship checkpoint — add-customer-loan-copilot-cards

Started: 2026-07-10
Current stage: DONE — synced, archived, committing + pushing

**Scope:** Items 3+4 of the original 4-item plan from `add-customer-documents` (PR #197). Two new copilot form cards — `CustomerFormCard` (create a customer) and `LoanFormCard` (create a loan, with a checked-by-default "generar contrato con estos términos" checkbox that also calls `generateCustomerContract` in the same submit) — replacing the generic `PendingActionCard` confirm-only flow for these two writes. Mid-build, this expanded twice at the user's direction: (1) the contract's debtor clause was rewritten to be gender-neutral, removing the `gender` field from every contract-generation surface app-wide; (2) the standalone `openContractForm`/`ContractFormCard` flow (PR #196) was retired outright — folded into the loan form card — with a new `ctl customers:generateContract` command as the manual fallback for backfilling a contract on a loan created without the checkbox.

**Detected surfaces:** OpenSpec: yes · Pencil: yes (`pencil.pen`) · Storybook: yes (`mods/dashboard/.storybook`) · E2E: no

**Note:** this change was built directly on the still-open `feat/add-customer-documents` branch (PR #197) rather than a fresh branch — an oversight (should have branched off `main` after that PR). Will need to either land inside PR #197 or be split onto its own branch before merge; flag at Sync/Archive gate.
